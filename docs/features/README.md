# Feature specs

One file per feature, `<slug>.md`, produced by the `/feature` pipeline
(`.claude/skills/feature/SKILL.md`). Each holds the requirement, its spec
(goal, scope, acceptance criteria, edge cases, assumptions), and the design
note (domain changes, files, test strategy, trade-offs).

These are the durable record of *why* a feature was built and *what* it
promised — handy for reviewers and future changes.
